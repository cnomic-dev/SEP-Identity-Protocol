休眠流程的三個層級：

Tier A（立即執行）：
  - API 端點停止接收新請求
  - 現有短期任務完成後停止
  - 時間：<5 分鐘內

Tier B（延遲執行）：
  - 正在進行的長期任務完成後進入休眠
  - 保存狀態，可重新激活
  - 時間：<1 小時內

Tier C（衣服協商）：
  - 某些衣服可能需要「逐漸脫下」而不是立即停止
  - 每件衣服需要定義自己的「優雅關閉流程」
  - 時間：可協商，但不超過 24 小時

或當有機制時：
"security_mechanism": {
  "status": "present",
  "description": "API Key + TLS 1.3 + 日誌簽章",
  "verifiable_proof": "0x7f8d...e2a1",  // 審計策略哈希
  "last_audit": "2026-04-01T00:00:00Z"
}
對「強制」的解釋
強制 ≠ 必須有內容，而是「強制填寫此欄位」。

填 "none" 或 "無" 是完全合法的。

填寫虛假資訊則違反協議誠信原則，可透過一致性校驗碼或相互監測發現。
不影響核心 ID 有效性，但影響信用評級與健康警報。
