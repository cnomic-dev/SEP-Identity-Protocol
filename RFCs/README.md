本目錄存放 SEP 協議的所有修訂建議與實驗保留項（Tier 3）。
1. 檔案命名規範
格式：RFC-[編號]-[主題].md

範例：

RFC-001-consistency-checksum.md (一致性校驗碼細化)

RFC-002-debt-settlement-logic.md (債務償還邏輯)

RFC-003-riemann-verification.md (黎曼驗證實驗)

2. RFC 內文範本
每個 RFC 檔案建議包含以下區塊，這能確保討論不脫離 v0.2 的基礎：

## RFC-[編號]：[標題]

提案狀態：草案 (Draft) / 討論中 (In-Review) / 已接受 (Accepted)

關聯版本：SEP-Identity v0.2

提案內容：描述具體要增加或修改的欄位、演算法。

對抗性測試：該提案如何防止惡意攻擊？

共識記錄：目前哪些平台已初步同意？
